Vue.js

A progressive framework for building user interfaces

Welcome everybody, thanks, etc.

  • I want to talk about a very cool Javascript framework for building interfaces
  • I've used it for a number of things - (DIA rule builder, large scale SPA)
  • I urge you not to regard Vue as this week's special. It's been around for a while and is child's play to learn

What is Vue?

It's a bit like our friends:

  • Knockout
  • React
  • Angular
  • etc

Vue is much like the other players in its core intentions.

  • You can build full on, balls out single page applications. Vue provides a routing engine and nice ways to manage complex state.
  • Or you can build snappy, extremely custom web controls or interface elements
  • Or you could make reusable components to use all over your app, or plug-in to other apps.

Some history:

  • Created by Evan You (from Google's Angular team)
  • Featured as a rising star on GitHub - Vue has clocked over twice as many stars as Angular 2

Why another JS framework?

  • This is a valid argument to have. i.e. is it worth learning a new framework when Angular 1.3 works just fine for me?
  • Users want snappy controls and more fluid interactions.
  • Implementing an ultra modern user experience is 100x easier with an ultra modern framework.

Why is Vue so popular?

https://10clouds.com/blog/vuejs-angular-react/
  • Fast
  • Reactive
  • Well-documented
  • Flexible and offers libraries
  • Utilises ES6 features
  • Reliable project generator
  • Supports hot reloading

Design qualities

  • Less opinionated
  • Incrementally adoptable
  • Fast and light
  • Scales up and down
  • Vue is approachable. Vue JS is just Javascript. There are no weird constructs or observables, it's just JS objects all the way down. When you deal with Vue, you're just writing plain old JS. Everything is observable.
  • For this reason, Vue is super easy to learn. Documentation states about a day for a developer to be able to build something non-trivial.
  • You can use Vue with or without jQuery, if you don't want routing, or vuex/redux state management, you don't have to have it. With Vue, pull in features as required.
  • Versatile. Vue scales up and down - meaning that you can build monolithic applications with it. Or you can drop a script tag in your markup, pull in Vue and write a small, custom component for your existing app.

Featuring:

  • Fast virtual DOM
  • Compose with components
  • Single file components
  • Recursive components
  • Typescript, if you want
  • CLI project generator
  • Dev tooling for your browser
  • It uses a virtual DOM, but you never need to worry about it - it's hidden from the developer (unless you want it). - you can leverage the virtual dom to have a render() method, bypassing the need for a HTML template. - this could enable usage of JSX or even serverside rendering.
  • A Vue application is comprised of components. Components each have a HTML template, javascript and CSS (optionally).
  • In my opinion, Vue components are best expressed in their single-file form. e.g. 1 component per file. But you can define string templates and do it other ways.
  • There's a command-line interface for setting up new projects. This takes care of your webpack, eslint-ing, dependency management and all that jazz.
  • Vue gives you a browser plugin which is fantastic for inspecting the state of your application. (look at this in the demo)

The best parts

  • Simple
  • Easy to learn
  • Nice developer UX
  • Again, Vue JS is just Javascript
  • All your of data is observable
  • It's easy and less opinionated about how your app should look
  • Simple examples in the documentation actually hold up when you use them in anger
  • 89% developer satisfaction rating
