What is Node.js

  • JavaScript runtime (Server-side)
  • built on Google Chrome's JavaScript Engine (V8 Engine)
  • Written in C/C++
  • Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine
  • Node.js was developed by Ryan Dahl in 2009
  • Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient
  • The Node.js project is supported by the Node.js Foundation

Why node.js

JSConf 2009: The first presentation on Node.js from Ryan Dahl

  • IO needs to be done differently
  • The event-loop approach: Apache vs Nginx
  • Threaded concurrency is a leaky abstraction.
  • Javascript designed speciﬁcally to be used with an event loop.
  • Many web applications have code that are just wait for response.
  • Better software can multitask. Other threads of execution can run while waiting => Is that the best that can be done? A look at Apache and NGINX.
  • Context switching is not free Execution stacks take up memory For massive concurrency, cannot use an OS thread for each connection.
  • Javascript designed speciﬁcally to be used with an event loop: Anonymous functions, closures. Only one callback at a time. I/O through DOM event callbacks.
  • The culture of Javascript is already geared towards evented programming.
  • This is the node.js project: To provide a purely evented, non-blocking infrastructure to script highly concurrent programs.
